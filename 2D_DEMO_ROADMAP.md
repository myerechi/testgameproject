# 2D 서바이버 데모 개발 로드맵

## 1. 목표 정의
- 현재 3D 임시 데모를 2D 탑다운 기반으로 전환
- "짧게 반복 플레이 가능한 데모"를 먼저 완성
- 이후 콘텐츠(무기/적/맵/메타 성장)를 점진적으로 확장

## 2. 최우선 전환 작업 (3D -> 2D)

### 2.1 렌더/씬 구조
- 2D 카메라(Orthographic) 기준으로 월드 스케일 고정
- SpriteRenderer 기반 캐릭터/적/투사체 표현
- Sorting Layer/Order 기준 정리 (`Background`, `Character`, `Projectile`, `UI`)
- 타일맵 또는 반복 배경 스프라이트로 2D 필드 구성

### 2.2 물리/충돌
- 3D Collider/Rigidbody 제거
- `Rigidbody2D`, `Collider2D`로 변경
- Layer Collision Matrix 설정:
- `Player`, `Enemy`, `Projectile`, `Pickup` 간 충돌 규칙 명확화
- 히트 판정 방식 통일:
- `OnTriggerEnter2D` 또는 수동 거리 판정 중 하나로 고정

### 2.3 입력/이동
- WASD/방향키 + 게임패드 스틱 지원 유지
- 2D 벡터 이동으로 단순화 (`Vector2`)
- 대각선 이동 속도 정규화

## 3. 코어 게임 루프 기능

### 3.1 플레이어 생존 루프
- 이동
- 자동 공격
- 적 웨이브 생존
- 경험치 획득
- 레벨업 선택
- 시간 경과에 따른 난이도 상승

### 3.2 타이머/난이도 곡선
- 0~3분: 튜토리얼 난이도
- 3~8분: 무리 스폰 증가
- 8분 이후: 엘리트/특수 적 등장
- 매 1분 단위로 "난이도 티어" 상승

### 3.3 승패 조건
- 기본 패배: HP 0
- 데모 승리 조건:
- 10분 생존 시 클리어 또는 미니보스 처치 시 클리어

## 4. 전투 시스템 상세

### 4.1 무기 시스템 (데모 1차)
- 기본 무기 3종:
- 직선 투사체(기본 총알)
- 근접 회전 무기(플레이어 주변)
- 관통 빔 또는 직선 관통탄
- 각 무기 공통 스탯:
- 공격력, 공격속도, 투사체 속도, 발사 수, 범위, 치명타 확률
- ScriptableObject 기반 무기 데이터화

### 4.2 무기 강화/진화
- 레벨업 선택 시 무기 강화 수치 상승
- 동일 무기 5레벨 도달 + 특정 패시브 보유 시 진화 해금
- 진화 무기는 이펙트/타격감 우선 개선

### 4.3 적 시스템
- 일반 적 아키타입 4종:
- 추적형(기본)
- 돌진형(짧은 스프린트)
- 원거리형(투사체 발사)
- 탱커형(고체력 저속)
- 엘리트 적:
- 고유 패턴 1개 + 처치 시 대량 경험치 드랍
- 미니보스:
- 1~2개 스킬 패턴, 명확한 텔레그래프 제공

### 4.4 피격/피해 처리
- 피격 무적 프레임(예: 0.3초)
- 넉백 여부를 무기별 옵션화
- 도트 데미지(중독/화상) 확장 가능 구조

## 5. 성장 시스템 상세

### 5.1 레벨업 선택 UI
- 레벨업 시 일시정지 + 3개 선택지 표시
- 선택지 분류:
- 신규 무기 획득
- 보유 무기 강화
- 패시브 강화
- 중복/비효율 선택지 방지 로직

### 5.2 패시브 아이템
- 데모 우선 패시브 6종:
- 최대 체력 증가
- 이동속도 증가
- 공격속도 증가
- 투사체 개수 증가
- 범위 증가
- 경험치 획득 범위 증가

### 5.3 드랍 시스템
- 일반 적: 소형 경험치 젬
- 엘리트: 대형 경험치 젬 + 회복 픽업 확률
- 미니보스: 보상 상자(랜덤 고등급 강화)

## 6. 맵/웨이브 콘텐츠

### 6.1 맵
- 데모 맵 2종:
- 평원(시야 명확)
- 유적(장애물 배치)
- 맵별 룰 1개:
- 스폰 가중치 또는 환경 효과

### 6.2 웨이브 이벤트
- 시간 기반 이벤트:
- 적 떼 러시
- 엘리트 경고 후 등장
- 보급 상자 드랍

## 7. UX/UI 기능

### 7.1 인게임 HUD
- HP, 레벨, 경험치 바, 생존 시간, 처치 수
- 현재 보유 무기/패시브 아이콘 표시
- 미니보스 등장 경고 UI

### 7.2 메뉴/흐름
- 타이틀 -> 캐릭터 선택 -> 맵 선택 -> 플레이 -> 결과 화면
- 결과 화면:
- 생존 시간, 처치 수, 획득 골드, 선택 빌드 요약

### 7.3 조작/접근성
- 키 리바인딩(최소 이동/일시정지)
- 화면 흔들림 강도 옵션
- 히트 이펙트 강도 옵션

## 8. 메타 진행 (데모 이후 확장)
- 런 종료 시 골드 획득
- 영구 강화 트리:
- 체력, 공격력, 리롤 횟수, 시작 무기 슬롯
- 해금 구조:
- 특정 도전과제 달성 시 무기/캐릭터 해금

## 9. 기술 구조 정리

### 9.1 데이터 주도 설계
- `ScriptableObject`:
- 무기 정의
- 적 정의
- 패시브 정의
- 웨이브 정의

### 9.2 매니저 분리
- `GameManager`: 상태 전이(타이틀/플레이/결과)
- `SpawnManager`: 웨이브/이벤트 스폰
- `CombatManager`: 데미지 계산/효과 적용
- `UIManager`: HUD/레벨업/결과 UI
- `AudioManager`: SFX/BGM 제어

### 9.3 성능 최적화
- 오브젝트 풀링:
- 투사체, 적, 경험치 젬, 이펙트
- GC 최소화:
- Update 할당 제거, 캐시 재사용
- 모바일 고려 시:
- 적 수 상한/LOD식 이펙트 단계 조절

## 10. 사운드/연출
- 타격음, 처치음, 레벨업 음향 추가
- 무기별 피격 이펙트(색상/스프라이트 플래시)
- 카메라 흔들림은 강도 제어 가능하게 구현
- 보스 등장 시 연출 컷인/경고 배너

## 11. 테스트 체크리스트
- 5분 이상 플레이 시 성능 저하 없는지
- 레벨업 선택지 논리 꼬임 없는지
- 무적 프레임/도트 데미지 중복 버그 없는지
- 게임오버/재시작 시 상태 초기화 정상인지
- 해상도(16:9, 21:9, 모바일 비율) UI 깨짐 없는지

## 12. 마일스톤 제안

### M1: 2D 기반 전환 (1주)
- 2D 씬/카메라/입력/충돌 전환
- 기본 적 1종 + 기본 무기 1종으로 플레이 가능

### M2: 데모 플레이어블 (1~2주)
- 무기 3종, 적 4종, 레벨업 UI, 엘리트 적
- 10분 생존 루프 완성

### M3: 데모 품질 업 (1주)
- 맵 2종, 결과 화면, 사운드/이펙트, 밸런싱
- 버그픽스 + 성능 최적화

### M4: 확장 준비 (선택)
- 메타 성장, 해금 시스템, 저장/불러오기
